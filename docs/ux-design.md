**Date:** 2025-12-07
**Version:** 1.0

## Design Principles

1. **One-Click Simplicity**: Streamers should go from landing to working overlay in < 2 minutes
2. **Channel-Centric Navigation**: Every URL makes the channel context obvious
3. **Familiar Patterns**: Use conventions streamers know from Twitch/YouTube dashboards
4. **Progressive Disclosure**: Start simple, reveal advanced features as needed

## Responsive Considerations

### Mobile Views

**Admin Dashboard on Mobile**
- Sidebar becomes hamburger menu
- Cards stack vertically
- Full-width forms

**Profile Page on Mobile**
- Single column layout
- Large touch targets for powerup selection
- Swipe-friendly voice selector

### Overlay (OBS Browser Source)
- Fixed dimensions (1920x1080 default)
- No responsive design needed
- Streamers configure in OBS

---

## Accessibility Requirements

1. **Keyboard Navigation**: All interactive elements focusable
2. **Color Contrast**: WCAG AA minimum (4.5:1 for text)
3. **Screen Reader**: All images have alt text, forms have labels
4. **Focus Indicators**: Visible focus states on all interactive elements
5. **Error Messages**: Clear, descriptive error messages associated with inputs
